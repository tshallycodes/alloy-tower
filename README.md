# AlloyTower Property Price Prediction

End-to-end machine learning system for predicting residential property sale prices, built for AlloyTower Inc.

---

## Project Overview

- **Dataset**: 2,090 cleaned property records across 33 US states
- **Target**: `last_sale_price` (USD)
- **Task**: Regression
- **Models**: Ridge Regression, Random Forest, XGBoost
- **Tracking**: MLflow with SHAP explainability
- **Deployment**: FastAPI (REST) + Streamlit (UI) + Docker

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Configure environment

Copy `.env` and fill in any placeholders (defaults work locally):

```bash
# .env
MLFLOW_TRACKING_URI=sqlite:///mlflow.db
BEST_MODEL=models/gb_model.pkl
API_URL=http://localhost:8000
GLOBAL_PRICE_MEAN=781993.0
```

## Running Locally

### Step 1 вЂ” Feature Engineering

Run from the project root:

```bash
python src/feature_engineering.py
```

This produces:
- `data/processed/ft_eng.csv`
- `models/feature_columns.json`
- `models/property_type_cols.json`
- `models/city_encoding.json`
- `models/state_encoding.json`

### Step 2 вЂ” Model Training

Open and run all cells in `src/model.ipynb` (run from the `src/` directory so relative paths resolve correctly):

```bash
cd src
jupyter notebook model.ipynb
```

Models are saved to `models/`. MLflow run data is stored in `src/mlruns/`.

### Step 3 вЂ” Start the API

```bash
uvicorn deployment.main:app --reload --port 8000
```

API docs available at [http://localhost:8000/docs](http://localhost:8000/docs).

### Step 4 вЂ” Start the Streamlit UI

```bash
streamlit run deployment/app.py
```

UI available at [http://localhost:8501](http://localhost:8501).

## Docker

### Build and run both services

```bash
docker-compose up --build
```

- API: [http://localhost:8000](http://localhost:8000) В· Docs: [http://localhost:8000/docs](http://localhost:8000/docs)
- Streamlit: [http://localhost:8501](http://localhost:8501)

The `models/` directory is bind-mounted into the API container, so swapping a model does not require a rebuild.

### Run services individually

```bash
# API only
docker build -t alloytower-api -f docker/Dockerfile.api .
docker run -p 8000:8000 --env-file .env -v $(pwd)/models:/app/models alloytower-api

# Streamlit only
docker build -t alloytower-ui -f docker/Dockerfile.streamlit .
docker run -p 8501:8501 --env-file .env -e API_URL=http://localhost:8000 alloytower-ui
```

## API Reference

### `POST /predict`

Predict the sale price for a property.

**Request body** (JSON):

| Field | Type | Description |
|---|---|---|
| `zip_code` | int | ZIP code |
| `latitude` | float | Decimal latitude |
| `longitude` | float | Decimal longitude |
| `bedrooms` | int | Number of bedrooms |
| `bathrooms` | float | Number of bathrooms |
| `sqft` | int | Interior square footage |
| `lot_size_sqft` | float | Lot size in sqft |
| `year_built` | int | Year of construction |
| `days_since_sale` | int | Days since last recorded sale |
| `tax_year` | int | Assessment tax year |
| `owner_occupied` | bool | Whether owner-occupied |
| `property_type` | string | One of: `Single Family`, `Townhouse`, `Condo`, `Multi Family` |
| `city` | string | City name |
| `state` | string | Two-letter state code (e.g. `AZ`) |

**Example request:**

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "zip_code": 85001,
    "latitude": 33.4484,
    "longitude": -112.074,
    "bedrooms": 3,
    "bathrooms": 2.0,
    "sqft": 1500,
    "lot_size_sqft": 5000,
    "year_built": 1990,
    "days_since_sale": 365,
    "tax_year": 2024,
    "owner_occupied": true,
    "property_type": "Single Family",
    "city": "Phoenix",
    "state": "AZ"
  }'
```

**Example response:**

```json
{ "predicted_price": 524300.75 }
```

Unrecognised cities or states fall back to the global mean price encoding (`GLOBAL_PRICE_MEAN` env var).

## MLflow

MLflow experiments are tracked under the `alloytower-predictions` experiment.

Start the MLflow UI (run from the `src/` directory):

```bash
cd src
mlflow ui --backend-store-uri sqlite:///mlflow.db --port 5000
```

Open [http://localhost:5000](http://localhost:5000) to compare runs, metrics (RВІ, RMSE, MedAE), parameters, and SHAP summary plots.

## Switching the Best Model

After reviewing the comparison table printed at the end of `model.ipynb`, update the `BEST_MODEL` variable in `.env`:

```bash
# Use XGBoost (default)
BEST_MODEL=models/gb_model.pkl

# Use Random Forest
BEST_MODEL=models/rf_model.pkl

# Use Ridge Regression
BEST_MODEL=models/ridge_model.pkl
```

**Local restart**: restart `uvicorn` after changing `.env`.

**Docker**: the `models/` directory is bind-mounted, so changing `.env` and restarting the container is sufficient вЂ” no image rebuild needed:

```bash
docker-compose restart api
```

## Feature Engineering Details

| Step | Action |
|---|---|
| Drop identifiers | `property_id`, `id`, `full_address`, `street_address`, `unit`, `owner_name`, `assessor_id`, `last_sale_date` |
| Drop direct leakage | `price_per_sqft` (derived from target) |
| Drop circular leakage | `assessed_value` (r=0.99), `annual_tax` (r=0.91) вЂ” correlation в‰Ґ 0.90 with target |
| Rare city grouping | Cities with <10 properties в†’ `'Other'` |
| Encoding | `property_type` в†’ one-hot (4 columns, no drop_first); `city`, `state` в†’ target encoding |
| Collinearity pruning | Drop one feature from any pair with Pearson r > 0.90 |
